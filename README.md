# main-cn-notes

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放各类独立的 HTML 页面，这些页面内容相互独立，不归属于任何特定域名或网站。每个页面可视为一份独立的笔记、工具页面或信息展示页，以 HTML 格式直接呈现。

仓库采用静态文件方式管理，不依赖后端服务，所有页面均可通过 GitHub Pages 直接访问。

## 目录说明

```
├── index.html          # 仓库首页（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
└── assets/             # 公共资源（CSS、JS、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：页面归档目录，每个 HTML 文件为一个独立页面。
- `assets/`：存放页面共用的样式、脚本或图片资源，避免重复。

## 页面归档说明

- 每个 HTML 页面均为自包含的独立文件，可脱离仓库环境在浏览器中直接打开。
- 页面之间无强关联，不共享状态或数据。
- 页面内容可能随时间更新或调整，历史版本可通过 Git 提交记录追溯。

## 维护说明

- 欢迎提交 Pull Request 添加新页面或改进现有页面。
- 请确保新增页面为独立的 HTML 文件，不依赖外部不可控资源。
- 若页面引用 `assets/` 中的资源，请使用相对路径。
- 不建议在页面中加入跟踪、统计或外部请求代码，保持页面简洁。
- 如需归档大型数据集或二进制文件，请考虑其他存储方式。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在本地打开 `pages/` 下的 HTML 文件。
3. 或通过 GitHub Pages 功能在线访问（需在仓库设置中开启 Pages 并选择根目录或 `/docs` 目录）。
